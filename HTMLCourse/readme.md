# Memo

重要なルール：H1は1ページに1つだけ
HTML上のルール違反ではありませんが
H1は1ページに1つだけにするのがベストプラクティスです
H2以降は複数使って構いません

## 太字と斜体

テキストを太字・斜体にする方法には古い書き方と新しい書き方があります。

- 用途古い書き方（非推奨）
  - <b>
  - <i>

- 新しい書き方（推奨）
  - <strong>
  - <em>（emphasize）

<strong>や<em>が推奨される理由はセマンティック（意味的）なHTMLの
考え方に よるものです

<b>や<i>は見た目を変えるだけですが
<strong>は「重要なテキスト」
<em>は「強調したいテキスト」という意味を持ちます

```
<strong>H</strong>yper<strong>T</strong>ext
<strong>M</strong>arkup <strong>L</strong>anguage
```

## 番号付きリスト

番号付きリストには<ol>（ordered list）要素を使います
リストの各項目には<li>（list item）要素を使います。

```
<ol>
  <li>項目1</li>
  <li>項目2</li>
  <li>項目3</li>
</ol>
```

## 箇条書きリスト（順序なしリスト）

箇条書きには<ul>（unordered list）要素を使います
項目の順序が重要でない場合に使います
中の各項目はやはり<li>要素を使います

```
<ul>
  <li>項目A</li>
  <li>項目B</li>
  <li>項目C</li>
</ul>
```

## 構造とセマンティックスの重要性

すべてのコンテンツを適切な要素で囲むことが重要です
HTMLは改行や空白を無視するため
要素なしでテキストをそのままbodyに書いてしまうと
ブラウザ上では意味のない一塊のテキストとして表示されてしまいます
各要素には意味があり、その意味に基づいてブラウザが正しく表示してくれます。

- 段落には<p>
- 番号付きリストには<ol>
- 箇条書きには<ul>
- リストの各項目には<li>

このようにコンテンツを論理的に構造化することが、良いHTMLの基本です。

## HTML画像要素と属性：要約

- 画像の挿入

画像には<img>要素を使います
閉じタグがなく、末尾に/を付ける自己完結型の要素です。

```
<img src="post-img.jpg" alt="画面上のHTMLコード" width="500" height="200" />
```

- 主な属性

src — 画像ファイルのパスを指定
alt — 画像の説明文。以下の理由から必ず書くこと

検索エンジンが画像の内容を把握できる
視覚障害者のスクリーンリーダーが読み上げてくれる
画像が読み込めない場合の代替テキストとして表示される

width / height — 画像のサイズをピクセルで指定
片方だけ指定すると縦横比が自動維持される

- その他の重要な属性

<html lang="en"> — ページの言語を指定（日本語ならja）
<meta charset="UTF-8"> — 文字コードを指定。headの中に書く

```
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <title>ページタイトル</title>
  </head>
</html>
```

ポイント：属性とは
属性とは要素を説明するためのデータです
属性名="値"の形式で開始タグの中に書きます

## HTMLリンク（アンカー要素）：要約

- 基本構文

リンクには<a>（anchor）要素を使います。

```
<a href="https://example.com">リンクのテキスト</a>
```

- リンクの2種類

外部リンク : 自分のサイト外のURLへのリンク

```
<a href="https://developer.mozilla.org">MDN web docs</a>
```

内部リンク : 自分のサイト内の別ページへのリンク

```
<a href="blog.html">ブログへ</a>
```

- 主な属性

- href — リンク先のURLまたはファイル名を指定
  これがないと見た目がただのテキストになりリンクにならない

- target="\_blank" — リンクを新しいタブで開く

- プレースホルダーリンク

URLがまだ決まっていない場合は`#`を使う
クリックするとページ上部に戻る

```
<a href="#">未定のリンク</a>
```

- ポイント

リンクはインターネットをウェブとして機能させる根本的な要素です
段落などの要素の中にリンクを埋め込むことも可能で
テキストの一部をリンクにできます

## HTML構造化要素：要約

ページのコンテンツを論理的にグループ化するためのHTML5要素を学びます
これらの要素は見た目は変えませんが、コンテンツに意味と構造を与えます

- 主な構造化要素
  - <nav>ナビゲーションリンクのまとめ
  - <header> ページやセクションの上部
  - <article>ブログ記事など独立したコンテンツ
  - <footer>ページ下部（著作権表示など）

- ページ全体の構造例

```
<body>
  <header>
    <nav>リンク集</nav>
    <h1>タイトル</h1>
  </header>

  <article>
    <header>記事の見出し・著者・画像</header>
    <p>本文...</p>
  </article>

  <footer>&copy; The Code Magazine</footer>
</body>
```

- ポイント

`<header>`はページ全体だけでなく
`<article>`の中など小さな単位にも使える
`&copy;`のようなHTMLエンティティを使うと©などの特殊文字を表示できる

これらの要素をきちんと使うことがセマンティックHTMLの考え方につながり
CSSでのスタイリングにも重要
